Taxonomy vs. Popularity in the birds of New Zealand

Posted by Amelia E. Barber on Sunday, November 24, 2019

Tidy Tuesday looking at association between bird phylogeny and popularity

The Tidy Tuesday dataset this week is on New Zealand’s “Bird of the Year 2019” so I thought it might be fun to look at how the taxonomy of the birds of NZ correlates with their popularity.

The packages we’ll be using this week, in addition to the core tidyverse packages, are wikitaxa to help convert common names to scientific names, treeio to read the phylogenetic tree into R, ggtree to visualize the phylogeny, and patchwork to combine the tree with a barchart of the vote data, so let’s load these and get started.

library(tidyverse)
library(wikitaxa)
library(treeio)
library(ggtree)
library(patchwork)

We can also grab the data from the Tidy Tuesday repo.

The “Bird of the Year” voting was done in a rather complicated instant runoff system over several weeksm, so if we look at the raw data, we can see that we have dates, hours, and vote ranks because people could vote for up to 5 birds. To simplify this, we’ll just look at the total number of votes because that give’s the same results at least for the top birds that we’re focusing on.

nz_bird <- readr::read_csv("https://raw.githubusercontent.com/rfordatascience/tidytuesday/master/data/2019/2019-11-19/nz_bird.csv")

Unfortunately, the dataset only provides us with the common names of the birds, while any sort of taxonomic database is going to require (or be more accurate) if we can provide the scientific names.

Use wikitaxa to obtain for scientific names from a bird’s common name

I initially tried to get the scientific names using the comm2sci() function from the taxize package, but the results were pretty incomplete and inaccurate, so I next tried the wikitaxa package which provides an easy way to access the WikiSpecies API.

bird_list <- top_birds %>% 
    set_names() %>% 
    map(wikitaxa::wt_wikispecies_search)

Pull results from list

query_results <- map(bird_list, ~pluck(.x, "query", "search"))

Manually add scientific name for missing birds because it is only 3. If part of a bigger project, would look into trying to webscrape latin names from wikipedia.

missing_bird_names <- tibble(

Save to file and then input to NCBI

Rather than building our own phylogeny from sequence data, we’re going to use the NCBI Common Tree to infer our phylogeny.

write_lines(plot_set, "sci_names.txt")

Import tree back into R

raw_tree <- treeio::read.tree("~/Downloads/phyliptree_fix.phy") %>% 
    as_tibble() %>% 
    mutate(label = str_replace_all(label, "_", " ")) %>% 
    left_join(bird_names_complete, by = c("label" = "latin_name")) %>%
    mutate(label = if_else(!is.na(common_name), glue::glue("{common_name} ({label})"),
                           label))

tree <- as.phylo(raw_tree)

ggtree(tree, branch.length = "none") +
    geom_nodelab(geom = "label", size = 3) +
    geom_tiplab() +
    coord_cartesian(clip = 'off') +
    ylim(0.5, 29.5) +
    theme_tree2() +
    theme(plot.margin=margin(6, 250, 6, 6),
          axis.line.x = element_line(color = "transparent"),
          axis.ticks.x = element_line(color = "transparent"),
          axis.text = element_text(color = "transparent"))

Now add vote data for each bird

bar_df <- nz_bird %>%
    filter(!is.na(vote_rank) & !is.na(bird_breed)) %>%
    count(bird_breed) %>%
    arrange(desc(n)) %>%
    top_n(30) %>% 
    left_join(bird_names_complete, by = c("bird_breed" = "common_name")) %>% 
    mutate(label = glue::glue("{bird_breed} ({latin_name})")) %>% 
    select(label, n)

highlight <- bar_df %>% 
    head(3) %>% 
    pull(label)

colors <- c("black", "darkcyan")

p <- ggtree(tree, branch.length = "none") +
    geom_nodelab(geom = "label", size = 3) +
    geom_tiplab(aes(color = label %in% highlight), show.legend = FALSE) +
    scale_color_manual(values = colors) +
    coord_cartesian(clip = 'off') +
    ylim(0.5, 29.5) +
